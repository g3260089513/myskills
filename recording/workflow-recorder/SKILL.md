# 工作流程记录器

在 Claude Code 会话中系统性地记录工作过程。追踪你做了什么、为什么这样做、遇到了什么问题、以及如何解决的。会话结束时生成一份完整的结构化报告，便于复盘和知识沉淀。

## 何时使用

以下情况应激活本技能：

- 用户说 "开始记录"、"记录工作流程"、"/workflow-recorder"
- 开始一个新功能开发或重构任务
- 调试复杂问题，需要追踪排查过程
- 任何希望事后回顾工作过程的会话
- 用户说 "生成报告" 或 "结束记录" 来触发报告输出

## 记录分类

本技能追踪三大类信息：

### 1. 工作流程总结
记录执行的每一步操作：探索了哪些文件、创建/修改了什么、运行了什么命令、测试结果如何。
- 格式：时间戳 + 步骤名称 + 执行内容 + 涉及文件 + 结果

### 2. 对话关键决策
记录与 AI 协作过程中做出的重要决定：为什么选择方案 A 而不是方案 B、架构决策的理由、技术选型的权衡。
- 格式：时间戳 + 决策主题 + 背景 + 候选方案 + 最终决策 + 理由

### 3. 问题与解决方案
记录遇到的每个问题和解决过程：错误信息、尝试过的方法、最终有效的解决方案、以及如何避免再次出现。
- 格式：问题编号 + 发现时间 + 问题描述 + 错误信息 + 尝试方案 + 最终解决 + 预防建议

## 工作流程

### 阶段一：开始记录

当用户激活技能时，执行以下操作：

1. **创建报告目录**：如果项目根目录下不存在 `reports/` 文件夹，则创建它。
2. **初始化报告文件**：创建 `reports/workflow-YYYY-MM-DD-HHmm.md`（使用当前时间戳）。
3. **写入报告头部**：填入项目名称、日期等基本信息。
4. **确认开始**：告知用户报告文件路径，并简要说明三大记录分类。

示例开场白：
```
📋 工作流程记录已启动
📁 报告文件: reports/workflow-2026-06-05-1430.md

我会记录以下内容：
  1. 工作流程 — 每一步操作和结果
  2. 关键决策 — 重要选择及其理由
  3. 问题与解决 — 遇到的问题和解决方案

随时可以说 "记录 [类别]: [内容]" 来手动添加记录。
完成后说 "生成报告" 来输出最终报告。
```

### 阶段二：持续记录

在会话进行中，每当发生以下情况时，自动追加到报告文件：

- 完成了某个文件/功能的编写
- 运行了测试或构建命令
- 做出了一个技术决策
- 遇到了错误并进行了排查
- 发现了一个值得注意的模式或经验

**追加时遵循原则：**
- 每条记录简洁明了（2-5 行）
- 使用 `### HH:mm - [标题]` 作为条目标题
- 涉及文件使用相对路径（可点击跳转）
- 问题记录要包含错误信息原文

### 阶段三：手动补充

用户可以随时手动触发记录。支持以下简写：

| 用户说 | 记录到 |
|--------|--------|
| "流程: [内容]" | 工作流程 |
| "决策: [内容]" | 关键决策 |
| "问题: [内容]" | 问题与解决方案 |
| "记录一下: [内容]" | Claude 判断最合适的分类 |

Claude 会基于内容自动判断应该放入哪个分类，并按照对应格式写入报告。

### 阶段四：生成报告

当用户说 "生成报告" 或 "结束记录" 时：

1. **补充总结**：在报告末尾添加 "总结与后续行动" 章节，包含：
   - 本次成果：完成了什么、产出了什么
   - 遗留问题：还有哪些未解决的问题
   - 后续步骤：接下来的行动计划
2. **最终检查**：确认三大分类都有内容，格式一致
3. **呈现报告**：告知用户报告文件路径，并简要概述内容
4. **可选操作**：询问是否需要将报告内容直接展示出来

## 文件组织

- **报告保存位置**: 当前项目根目录下的 `reports/` 文件夹
- **文件命名规则**: `workflow-YYYY-MM-DD-HHmm.md`
- **如果 `reports/` 目录不存在**: 在阶段一自动创建
- **建议**: 将 `reports/` 加入 `.gitignore`（如果报告内容涉及敏感信息）

## 注意事项

1. **简洁优先**: 每条记录控制在 2-5 行，不要长篇大论。本技能的目标是记录要点，不是写日记。
2. **及时记录**: 遇到问题立刻记，不要等。事后回忆往往会遗漏关键细节（尤其是错误信息）。
3. **错误信息必贴原文**: 不要自己总结错误信息，直接粘贴原文，方便以后搜索。
4. **涉及文件用相对路径**: 使用 `src/foo/bar.ts:42` 格式，点击可跳转。
5. **报告写到磁盘不要常驻上下文**: 报告内容不需要反复读回，只在最终生成时读取一次。
