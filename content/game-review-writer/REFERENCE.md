# 游戏评测写作参考

> 此文件为 `game-review-writer` 的格式与风格参考。修改此文件即可调整输出格式，无需改动 SKILL.md。

---

## 1. 章节模板（深度分析型）

```
## 整体印象

[一段话定调：游戏类型 + 一句话核心论点。直接表态，不要迂回。]

## [核心系统1]：[论点式小标题]

[逐层拆解该系统——设计意图 → 实际表现 → 问题所在（如有）。用加粗强调关键概念。]

## [核心系统2]：[论点式小标题]

[同上结构。每个核心系统独立成节，深度展开。]

## 亮点与缺陷

[可合为一节或分拆为两节。列举具体设计层面，不要泛泛而谈"画面好/优化差"。]

## 玩家社区反馈

[引用搜索到的玩家评价——Steam评论、NGA讨论、Reddit帖子等。用社区声音验证或挑战你的个人判断。标注来源类型。]

## 总结与评分

[回到开篇的核心论点。给出最终评分（0-10，一位小数），可提及评分变化趋势。以一句有力的总结收尾。]
```

### 章节使用规则

- **必须保留**：整体印象、总结与评分（每篇评测的开头和结尾）
- **可按需增删**：中间的分析章节——取决于游戏有多少值得深度拆解的系统
- **"玩家社区反馈"节**：如果网络研究没找到足够有价值的社区讨论，可简化为正文中的一两句引用，不必独立成节
- **表格**：当需要并排对比（版本对比、武器对比等）时使用 Markdown 表格
- **分割线**：大版本更新或话题转换用 `---` 分隔

## 2. 评分参照表

| 分数 | 标签 | 含义 |
|------|------|------|
| 9.0-10 | 神作 | 类型标杆，几乎没有硬伤 |
| 8.0-8.9 | 优秀 | 品质出众，有几处可改进 |
| 7.0-7.9 | 佳作 | 整体不错，但有明显短板 |
| 6.0-6.9 | 勉强推荐 | 核心体验还行，但问题较多 |
| 5.0-5.9 | 不推荐 | 核心系统有严重缺陷 |
| <5.0 | 避雷 | 各方面都不行 |

评分使用**一位小数**（如 7.5、8.3、6.0）。如果评测是对同一游戏的更新复盘，可在总结中提到评分变化（如"从 S1 的 6.5 到 S2 的 6.0"）。

## 3. Frontmatter 字段参考

### 必填字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `gameName` | string | 游戏中文名 | `"逆战：未来"` |
| `developer` | string | 开发商名称 | `"琳琅天上"` |
| `releaseYear` | number | 发行年份（裸数字） | `2026` |
| `platform` | string[] | 平台列表 | `["PC", "PS5", "Xbox Series X"]` |
| `genre` | string[] | 类型标签（可多选） | `["FPS", "PVE"]` |
| `rating` | number | 评分 0-10（一位小数） | `8.5` |
| `playTime` | string | 游玩时长 | `"350小时"` |
| `coverImage` | string | 封面图路径 | `"/images/game-reviews/elden-ring/cover.jpg"` |
| `tags` | string[] | 展示标签（3-6 个） | `["动作", "共斗", "数值设计"]` |
| `summary` | string | 一句话总结（卡片显示用） | 参见现有评测的 summary 写法 |
| `publishedAt` | string | 发布日期 YYYY-MM-DD | `"2026-01-13"` |

### 可选字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `status` | string | — | `"ongoing"`（游玩中，绿色徽章）/ `"completed"` / `"maintained"`（持续更新，金色徽章）|
| `featured` | boolean | `false` | 设为 `true` 可在首页精选展示 |
| `draft` | boolean | `false` | 设为 `true` 隐藏文章（不在列表页显示） |
| `updatedAt` | string | — | 最后更新日期 YYYY-MM-DD（评测更新时必须填写） |

### 路径约定

- `coverImage`: `/images/game-reviews/{slug}/cover.jpg`
- 优先使用 WebP/AVIF，JPG 作为后备

## 4. 写作风格指南

### 总体原则

- **中文撰写**，游戏名在正文中可用英文原名
- **第一人称**视角（"我"而非"笔者"）
- **断言式开头**——开门见山给出核心论点，不要铺垫
- **分析而非叙述**——评测的价值在于"为什么好/不好"，而非"有什么"
- **精准术语**——使用游戏设计术语（数值策划、成长曲线、反馈周期、心流等），不打引号回避

### 段落节奏

- 先抛结论，再展开论证（倒金字塔）
- 每个自然段聚焦一个论点
- 关键句用 **加粗** 突出，方便扫读

### 表格使用

当需要并排对比两个版本、多把武器、多个维度时，用 Markdown 表格：

```md
| 维度 | 状态 | 变化 |
|------|------|------|
| 武器平衡 | 两三把垄断 | → |
```

### 社区引用

引用玩家社区评价时，标注来源类型（Steam 评价、NGA 讨论、Reddit 帖子等），不要笼统说"很多玩家认为"。直接引语用引号包裹。

### 评分论述

评分不是贴标签，而是论点的自然收束。在总结节中解释为什么是这个分数——引述前文分析过的具体优劣，给出可量化的理由。

### 禁止模式

- ❌ 不要写"总的来说这款游戏还是不错的"（口水话，无信息量）
- ❌ 不要堆砌形容词不做分析（"精彩的战斗、美丽的画面、丰富的系统"）
- ❌ 不要用"笔者认为/个人觉得"类套话（直接说观点即可）
- ❌ 不要照搬网络评测的原话（用自己的语言整合）
- ❌ 不要回避批评——如果游戏有问题，明确指出，不要用"还需要打磨"代替具体批评

## 5. 现有评测参考

以下是从现有评测中提取的代表性段落，展示了深度分析风格的标志性写法。

### 开头定调

> 《逆战：未来》是琳琅天上对经典 IP 的一次现代化重启。游戏以 PVE 猎场刷图为核心，支持 PC 与移动端跨平台联机，承诺"不卖数值"走外观付费路线。公测至今 350 小时的体验下来，它的底子不差——枪感扎实、跨端体验流畅、刷图爽感在线。但深入游玩后，一系列设计层面的结构性缺陷逐渐浮出水面。

特点：游戏定位 + 个人时长 = 可信度 → "但"转折带出核心论点

### 系统拆解

> 武器插件分为四个位置——弹夹、充能、增伤、特殊效果。这套分层架构在理论上非常漂亮：前三位构成数值锚点，四号位负责机制创新，策划可以通过微调插件数值而非武器本体来实现平衡。但落地效果远未达到预期。

特点：先描述设计意图（为什么这样设计），再用"但"指出现实落差

### 核心原则

> **刷子游戏的第一原理是：单局收益预期必须与投入正相关，正向反馈周期不能过长。**

特点：从具体问题中提炼通用原则，加粗标出

### 结尾收束

> 《怪物猎人：世界》教会我一个道理：**优秀的数值策划不是堆砌数字，而是设计体验**。从装备阈值的临界点，到技能系统的分层引导，再到手部乘区与怪物模组的动态耦合——每一处都体现着"以玩家感受为中心"的刻意为之。

特点：回到个人体验，用加粗核心结论收束，列举前文分析过的关键概念形成回环
